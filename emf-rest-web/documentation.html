The generated API

EMF-REST generates a web application which is called as your Eclipse project. Inside the application, a servlet mapping called app is created which interprets the rest of the call. For instance, in our example the project is called Demo so the application will have the prefix http://<server>/Demo/app/<rest of the call>.

The servlet takes the url to navigate through the models and their elements. Thus, the next element in the call is the name of the model (e.g., Family in our example, i.e., http://<server>/Demo/app/<Family>). Once the model name has been indicated, you can ask for a particular model, e.g., the Simpsons family by adding at the end of the call (e.g,. http://<server>/Demo/app/Family/Simpsons). This last call can be seen as the entrypoint for your model and will return the data of the root element of your model.

With the previous entrypoint call, you can nagivate through your model using the reference names. For instance, the concept Family has the reference sons so we can call http://<server>/Demo/app/Family/Simpsons/sons to get the list of sons of the Simpsons family. Other example would be http://<server>/Demo/app/Family/Simpsons/parents, which returns the set of parents of the Simpsons family.

JSON structure

The elements of your model are automatically translated into a JSON object. A model elements is therefore represented by a JSON object which includes a set of name/value pairs representing its attributes/references. Attributes are represented as primitive-typed name/value pairs wheres as references are translated into a name/value pair where the value is an array of objects.

The generated Javascript API

For those of you that do not want to play with URLs and so on, EMF-REST also generates a Javascript file including a set of helper functions which helps you to do all the work to interact with the API.
