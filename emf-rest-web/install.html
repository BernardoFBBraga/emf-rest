Installation

How to install the Eclipse Plugin

To install/use EMF-REST, we recommend the J2EE distribution of Eclipse, that can be found here. EMF-REST has been tested with the Eclipse Kepler version.

Once your Eclipse is ready, you can proceed with the installation of EMF-REST by

  1. Clicking on Help / Install New Software...
  2. Adding a new Software Site. Click on Add... and fill in the form as indicated:
    (the update site is http://emf-rest.com/update/site.xml)
  3. You should now see the EMF-REST plugin in the list of components to be installed. Select it and click on Next:
  4. You can now quickly accept the rest of the steps (license, etc...) and reboot Eclipse

How to use the plugin

To generate a REST API for your models simply:

  1. Create a new Dynamic Web Project
  2. Create a new folder in the project, e.g., called model and copy your EMF model and their instances
  3. Right click on your EMF model (the file with extension .ecore and select the menu entrance EMF-REST / Generate All + Copy Libraries
  4. The generator will create all the needed infrastructure for your model
  5. Edit the variable called modelsPath of the class webmapi.storage.MemoryStorage to point at the folder containing your models
  6. The project is ready to be deployed into your application server

Running your generated application

The fastest way to try EMF-REST is to deploy it into an application server compatible with Eclipse. These steps will describe how to do it in Apache Tommee.

  1. Download Apache Tomee JAX-RS and unzip in your hard disk
  2. Run your recently created application by clicking on Run As / Run On Server
  3. Define a new server of type Tomcat v7.0 Server and configure the runtime environment to use the previously downloaded Apache Tomee
  4. Click on finish, the application will be deployed in the server
Remember!

Note that the default landing page of your EMF-REST generated application is not included in the generation process so it is normal no get an error when accessing to the root of the application. To check that everything is working fine remember the format used to navigate through the models (explained here).

Just as a quick reminder: http://<server>/<project-name>/app/<model-name>/<concrete-model-name>
